```javascript
  // CSP: dev keeps 'unsafe-inline' for Tailwind v4 HMR; prod is strict
  session.defaultSession.webRequest.onHeadersReceived((details, callback) => {
    const styleUnsafe = isDev ? " 'unsafe-inline'" : "";
    callback({
      responseHeaders: {
        ...details.responseHeaders,
        'Content-Security-Policy': [
          "default-src 'self'; " +
          "script-src 'self' 'unsafe-inline'; " +
          `style-src 'self'${styleUnsafe}; ` +
          `style-src-elem 'self'${styleUnsafe}; ` +
          "font-src 'self' data:; " +
          "img-src 'self' data: https:; " +
          "connect-src 'self' http://localhost:* ws://localhost:* https://www.googleapis.com https://oauth2.googleapis.com"
        ]
      }
    });
  });
```

**Changes from current CSP:**
1. **Removed** `https://fonts.googleapis.com` from `style-src` — no more CDN font import
2. **Removed** `https://fonts.googleapis.com` from `style-src-elem` — same reason
3. **Removed** `https://fonts.gstatic.com` from `font-src` — Inter is now bundled locally
4. **Added** `data:` to `font-src` — for any base64-embedded font data from fontsource
5. **Added** `const styleUnsafe = isDev ? " 'unsafe-inline'" : "";` — dev/prod conditional
6. **Changed** `style-src` to use template literal with `${styleUnsafe}` — only adds `'unsafe-inline'` in dev mode
7. **Changed** `style-src-elem` to use template literal with `${styleUnsafe}` — same dev/prod split
8. **Added** CSP comment for future maintainers

**Do NOT change** `script-src`, `img-src`, `connect-src`, or `default-src` — those are unchanged.
**Do NOT change** any code outside the CSP block (lines 1-91 and 108+ remain untouched).
