## Directory Purposes

**`src/`:**
- Purpose: React application source code (web SPA)
- Contains: Components, hooks, utilities, constants, styles
- Key files: `App.jsx` (main state container), `main.jsx` (entry point)

**`src/components/`:**
- Purpose: Reusable React components
- Contains: Presentational components (StatCard, Toast, buttons), modal dialogs
- Key files: `modals/` subdirectory contains complex dialogs

**`src/components/modals/`:**
- Purpose: Modal dialog components for user interactions
- Contains: Import wizard, conflict resolver, category manager, sync settings
- Pattern: Each modal is a controlled component receiving `onConfirm`/`onCancel` props

**`src/hooks/`:**
- Purpose: Custom React hooks
- Contains: `useGoogleDrive` for Electron environment integration
- Key behavior: Detects Electron availability, manages OAuth auth state, handles Drive operations

**`src/utils/`:**
- Purpose: Pure utility functions
- Contains: `formatCurrency`, `parseDate`, `parseAmount`, `categorize`, `findMatchingCategories`, `generateId`
- Usage: No side effects, used throughout app for data transformation

**`src/constants/`:**
- Purpose: Application-wide constants
- Contains: Color palette, built-in import profiles (Illimity, Fineco, generic), default categories with keywords
- Key: `DEFAULT_CATEGORIES` maps category names to keyword lists for auto-detection

**`electron/`:**
- Purpose: Electron main process and desktop integration
- Contains: Window creation, IPC handlers, Google Drive API client, auto-backup logic
- Key files: `main.cjs` (app lifecycle), `preload.cjs` (safe API exposure), `googleDrive.cjs` (Drive integration)

**`public/`:**
- Purpose: Static assets bundled with app
- Contains: Logo icon and images
- Usage: Referenced in HTML and Electron window configuration

**`dist/`:**
- Purpose: Built JavaScript/CSS output from Vite (generated)
- Contains: Minified React app, consumed by Electron in production

**`release/`:**
- Purpose: Electron packaged installers (generated by electron-builder)
- Contains: `.exe` portable installer, NSIS installer for Windows

## Naming Conventions

**Files:**
- Components: PascalCase with `.jsx` extension (`StatCard.jsx`, `ConfirmModal.jsx`)
- Hooks: camelCase with `use` prefix and `.js` extension (`useGoogleDrive.js`)
- Utilities: camelCase with `.js` extension
- Constants: camelCase with `.js` extension
- Main process: `.cjs` extension (CommonJS, not ES modules)

**Directories:**
- Component folders: plural lowercase (`components/`, `hooks/`, `modals/`)
- Feature directories: descriptive lowercase names

**Variables/Functions:**
- React state: camelCase (`transactions`, `selectedMonth`)
- State setters: `set` + PascalCase (`setTransactions`, `setSelectedMonth`)
- Event handlers: `handle` + action (`handleFile`, `handleWizardConfirm`)
- Callbacks: `on` + event (`onConfirm`, `onCancel`)
- Utility functions: descriptive verbs in camelCase (`formatCurrency`, `parseDate`, `categorize`)

**CSS Classes:**
- BEM-adjacent pattern: `block-element` (e.g., `.transaction-item`, `.modal-overlay`, `.btn-primary`)
- State modifiers: `.active`, `.disabled` (e.g., `.dropdown.active`)
- Semantic naming tied to structure

**Constants:**
- SCREAMING_SNAKE_CASE for truly constant values (`COLORS`, `ITEMS_PER_PAGE`, `MONTHS_IT`)
- PascalCase for object keys in import profiles (`DEFAULT_CATEGORIES`, `BUILTIN_IMPORT_PROFILES`)

## Where to Add New Code

**New Feature (e.g., new view/page):**
- Primary code: `src/App.jsx` (add state + handlers + view logic)
- Component files: `src/components/FeatureName.jsx`
- Modal dialogs: `src/components/modals/FeatureModal.jsx`
- Styles: Add rules to `src/App.css` with feature-specific class names
- Constants: Add to `src/constants/index.js` if feature needs shared config

**New Component/Module:**
- Implementation: `src/components/ComponentName.jsx`
- Export: Add to barrel file `src/components/index.js`
- Props type hints: Document in JSDoc comment above component definition
- Styles: Class names follow `component-element` BEM pattern in `src/App.css`

**Utilities:**
- Shared helpers: `src/utils/index.js`
- Export at module level: `export const functionName = ...`
- Always pure functions: no side effects, no external dependencies
- JSDoc comment above function definition with `@param` and `@returns`

**New Modal Dialog:**
- File: `src/components/modals/ActionModal.jsx`
- Export: Add to `src/components/modals/index.js`
- Pattern: Accept `onConfirm`, `onCancel` props + data props
- Structure:
  ```jsx
  export default function ActionModal({ title, data, onConfirm, onCancel }) {
    return (
      <div className="modal-overlay" onClick={onCancel}>
        <div className="modal" onClick={e => e.stopPropagation()}>
          {/* content */}
        </div>
      </div>
    );
  }
  ```

**Electron Integration:**
- Main process handlers: `electron/main.cjs` (add `ipcMain.handle(...)`)
- Preload API: `electron/preload.cjs` (add to `contextBridge.exposeInMainWorld`)
- Hook integration: `src/hooks/useGoogleDrive.js` (add state + IPC calls)

**New Import Profile:**
- Add to `BUILTIN_IMPORT_PROFILES` object in `src/constants/index.js`
- Schema: `{ name, headerRow, dateColumn, descriptionColumn, amountType, amountColumn?, incomeColumn?, expenseColumn?, idColumn? }`

**New Default Category:**
- Add to `DEFAULT_CATEGORIES` object in `src/constants/index.js`
- Format: `'Category Name': ['KEYWORD1', 'KEYWORD2', ...]`
- Keywords should be uppercase (matching is case-insensitive)
