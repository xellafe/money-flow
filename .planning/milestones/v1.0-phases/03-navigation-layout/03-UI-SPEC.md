## Animation Specification

All animations use Framer Motion v12 (import from `'framer-motion'`).

| Element | Property | From | To | Duration | Easing |
|---------|----------|------|----|----------|--------|
| Sidebar width | `animate={{ width }}` | 240px | 64px (collapse) | 200ms | `easeInOut` |
| Main content reflow | `layout` prop | flex-1 fills old space | flex-1 fills new space | 200ms | `easeInOut` |
| Chevron icon | `animate={{ rotate }}` | 0° | 180° (collapse) | 200ms | `easeInOut` |
| Nav item hover | CSS `transition-colors` | — | — | 150ms | default |
| Toggle/CTA button hover | CSS `transition-colors` | — | — | 150ms | default |

**Framer Motion config (use exactly):**
```js
// Sidebar width
transition={{ duration: 0.2, ease: 'easeInOut' }}

// Main content layout reflow
transition={{ layout: { duration: 0.2, ease: 'easeInOut' } }}

// Chevron rotation (same transition object)
transition={{ duration: 0.2, ease: 'easeInOut' }}
```

**No `LayoutGroup` needed** — sidebar and main content are siblings in the same render tree.

**Anti-patterns (must avoid):**
- Do NOT add `layout` prop to elements inside the sidebar (only the main content sibling uses `layout`)
- Do NOT combine CSS `transition-width` with Framer Motion `animate={{ width }}` on the same element
- Do NOT use `position: fixed` for sidebar (use flex row layout)

All icons use `className="shrink-0"` to prevent compression in flex containers.

**Note on nav item `aria-label`:** The `aria-label` must be present on the `<button>` element at all times (not just when collapsed) so screen readers announce the destination consistently regardless of sidebar state. When the label `<span>` is conditionally hidden (`!collapsed && <span>`), `aria-label` becomes the sole accessible name.
