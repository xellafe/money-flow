# Phase 6 — UI Design Contract: Modals Redesign

> Scope: ModalShell shared wrapper + 7 modal migrations (ConfirmModal, CategoryManager, SyncSettings,
> ImportWizard, ConflictResolver, CategoryConflictResolver, PayPalEnrichWizard) + SettingsView buttons.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in Phase 6                                       |
|-------|-------|--------------------------------------------------------|
| xs    | 4px   | Icon-to-label gap in buttons, tight inline spacing     |
| sm    | 8px   | Form field internal padding, button row gaps           |
| md    | 16px  | Modal body padding (all sides), form group spacing     |
| lg    | 24px  | Modal header padding, section breaks within long forms |
| xl    | 32px  | Not used in modal context                              |
| 2xl   | 48px  | Not used in modal context                              |
| 3xl   | 64px  | Not used in modal context                              |

**Modal-specific:**
- Modal panel padding: `p-6` (24px) header, `p-6` body
- Modal body max-height: `max-h-[80vh]` with `overflow-y-auto`
- Modal size sm: `max-w-md` — ConfirmModal, CategoryConflictResolver
- Modal size lg: `max-w-2xl` — ImportWizard, SyncSettings, CategoryManager, PayPalEnrichWizard, ConflictResolver
- Button padding: `px-4 py-2` (16px / 8px)

Exceptions:
- Touch targets: N/A (Electron desktop, no touch)
- ModalShell X button: `p-2` (8px) for adequate click area, `rounded-lg`

## Component Contracts

### ModalShell

```
File: src/components/ui/ModalShell.jsx
API: <ModalShell title={string} onClose={fn} size="sm|lg">{children}</ModalShell>
```

**Primary visual anchor:** `Dialog.Title` (18px / weight 600) — rendered as the first visible element in the modal header row; the first element the user's eye lands on when a modal opens.

**Structure:**
```
Dialog.Root open={true} onOpenChange → onClose
  Dialog.Portal
    Dialog.Overlay asChild → motion.div  (overlay layer, z-40)
    Dialog.Content asChild → motion.div  (panel, z-50, centered via fixed inset-0 flex items-center justify-center)
      header row: h-[60px] px-6 flex items-center justify-between border-b border-gray-200
        Dialog.Title: text-[18px] font-semibold text-gray-800
        Dialog.Close asChild → X button: p-2 rounded-lg text-gray-400 hover:text-gray-600 hover:bg-gray-100
      body: overflow-y-auto max-h-[80vh] p-6
        {children}
```

**Overlay animation (Framer Motion):**
```
initial: { opacity: 0 }
animate: { opacity: 1, transition: { duration: 0.2, ease: 'easeOut' } }
exit:    { opacity: 0, transition: { duration: 0.15, ease: 'easeIn' } }
```

**Panel animation (Framer Motion):**
```
initial: { opacity: 0, scale: 0.95 }
animate: { opacity: 1, scale: 1,    transition: { duration: 0.2, ease: 'easeOut' } }
exit:    { opacity: 0, scale: 0.95, transition: { duration: 0.15, ease: 'easeIn' } }
```

**App.jsx AnimatePresence wrappers:**
Each conditional modal render in App.jsx is wrapped with `<AnimatePresence>` — ModalShell renders `open={true}` always, AnimatePresence handles mount/unmount transitions.

### Button Variants (MOD-06)

All buttons share base: `rounded-lg px-4 py-2 text-sm font-semibold transition-colors`

| Variant     | Class                                       | Reserved For                              |
|-------------|---------------------------------------------|-------------------------------------------|
| Primary     | `bg-brand-600 hover:bg-brand-700 text-white` | Confirm, Save, Import, Apply, Connect     |
| Secondary   | `bg-gray-100 hover:bg-gray-200 text-gray-700` | Cancel, Back, Skip, Close               |
| Destructive | `bg-red-600 hover:bg-red-700 text-white`    | Delete, Remove — only after confirmation  |

**Button placement:** Each modal manages its own footer/action row at the bottom of `{children}` (no ModalShell footer slot).

### PayPalEnrichWizard Step Transitions

Internal step animation (horizontal slide via AnimatePresence):
```
outgoing: { x: -30, opacity: 0, transition: { duration: 0.2, ease: 'easeOut' } }
incoming: { x: 30 → 0, opacity: 0 → 1, transition: { duration: 0.2, ease: 'easeOut' } }
back direction: reverses sign (x: +30 out, x: -30 in)
```

No step progress indicator (decided in CONTEXT.md — content flows vertically).

### ConflictResolver & CategoryConflictResolver Footer Pattern

Both modals have multi-action footers. Declared layout:
- `ConflictResolver`: footer row with **Mantieni originale** (secondary), **Usa nuovo** (primary), **Salta** (secondary) — left-aligned skip, right-aligned primary pair
- `CategoryConflictResolver`: footer row with **Annulla** (secondary), **Applica** (primary)

## Interaction States

| State       | Specification                                                              |
|-------------|----------------------------------------------------------------------------|
| Hover       | All buttons: `transition-colors` 150ms (via Tailwind default duration)     |
| Focus       | `focus-visible:outline-2 outline-brand-500 outline-offset-2` (global rule already in index.css) |
| Disabled    | `opacity-50 cursor-not-allowed` — apply to primary CTA during async ops   |
| Loading     | Primary button shows spinner (Loader2 from lucide-react, `animate-spin`) + disabled state |
| ESC close   | Radix Dialog native — fires onOpenChange(false) → onClose                  |
| Backdrop click | Radix Dialog native — fires onOpenChange(false) → onClose              |
| Focus trap  | Radix Dialog.Content built-in — focus cycles within modal, restored on close |
