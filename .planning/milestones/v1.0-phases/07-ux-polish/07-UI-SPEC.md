> Visual and interaction contract for Phase 7. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> cursor sweep (UX-01 + UX-06), and page-transition fade (UX-07).

## Spacing Scale

Declared values (multiples of 4) — consistent with Phases 4–6:

| Token | Value | Usage in Phase 7                                              |
|-------|-------|---------------------------------------------------------------|
| xs    | 4px   | Icon-to-label gap inside Toast, button inner gaps             |
| sm    | 8px   | Toast internal padding (vertical), button padding (py-2)      |
| md    | 16px  | Toast internal padding (horizontal px-4), modal body padding  |
| lg    | 24px  | Toast position from viewport edge (bottom-6 right-6 = 24px)  |
| xl    | 32px  | Empty state vertical rhythm (py-16 = 64px top/bottom = 4×xl) |
| 2xl   | 48px  | Not used                                                      |
| 3xl   | 64px  | Empty state icon size (size=64)                               |

**Component-specific:**
- Toast: `px-4 py-3` (16px / 12px) — tight notification card
- Toast position: `fixed bottom-6 right-6 z-50` (24px from each edge)
- AddTransactionModal: `ModalShell size="sm"` → `max-w-md` (from Phase 6 contract)
- AddTransactionModal footer button padding: `px-4 py-2` (16px / 8px)
- Empty state icon: `size={64}` (Lucide numeric prop, not Tailwind class)
- Empty state text container: `max-w-xs` (20rem)
- Skeleton row: `px-4 py-3` — matches TransactionRow grid layout exactly

Exceptions:
- Touch targets: N/A (Electron desktop, no touch interaction)
- Header "Aggiungi" button: `px-4 py-2` — consistent with Phase 6 button base

## Component Contracts

### Toast (rewrite — UX-04)

```
File: src/components/Toast.jsx
Renders via: App.jsx wrapped in <AnimatePresence mode="wait"> keyed by toast message
```

**Visual structure:**
```
motion.div [fixed bottom-6 right-6 z-50]
  div [bg-white border border-gray-200 shadow-lg rounded-xl px-4 py-3 flex items-center gap-3]
    icon [size=18]                ← Check (text-green-500) or AlertCircle (text-red-500)
    span [text-sm text-gray-800]  ← message text
    button [ml-auto text-gray-400 hover:text-gray-600 transition-colors cursor-pointer]
      X [size=16]                 ← close button
```

**Framer Motion animation:**
```js
// Enter (300ms ease-out — matches UX-04 spec)
initial:  { opacity: 0, y: 20 }
animate:  { opacity: 1, y: 0,  transition: { duration: 0.3, ease: 'easeOut' } }

// Exit (200ms ease-in)
exit:     { opacity: 0, y: 20, transition: { duration: 0.2, ease: 'easeIn' } }
```

**App.jsx wiring:**
```jsx
<AnimatePresence mode="wait">
  {toast && (
    <Toast
      key={toast.timestamp ?? toast.message}
      message={toast.message}
      type={toast.type}
      onClose={hideToast}
    />
  )}
</AnimatePresence>
```

**Stacking:** Single toast at a time. `mode="wait"` ensures old toast exits before new one enters.
**Auto-dismiss:** 3000ms timeout inside Toast via `useEffect` — preserve existing behavior.
**Old CSS cleanup:** Remove `.toast`, `.toast.success`, `.toast.error` rules from `src/index.css` / `src/App.css` after migration.

### SkeletonTransactionRow (new — UX-02)

```
File: src/components/transactions/SkeletonTransactionRow.jsx
Usage: Render 5× in TransactionsView during isLoading=true state
```

**Visual structure — mirrors TransactionRow grid exactly:**
```
div [grid grid-cols-[1fr_120px_40px] px-4 py-3 items-center border-b border-gray-100]
  Col 1 [flex flex-col gap-1]
    div [animate-pulse bg-gray-200 rounded h-3 w-24]     ← date placeholder
    div [animate-pulse bg-gray-200 rounded h-4 w-48 mt-1] ← description placeholder
    div [animate-pulse bg-gray-200 rounded h-5 w-20 mt-1] ← category badge placeholder
  Col 2 [flex justify-end]
    div [animate-pulse bg-gray-200 rounded h-4 w-16]     ← amount placeholder
  Col 3 [w-10]                                            ← empty (no delete shimmer)
```

**aria attributes:** `aria-busy="true"` on the wrapper div. `aria-label="Caricamento transazioni"` on the first row (or on wrapper in TransactionsView).

**State management in TransactionsView (local state):**
```js
const [isLoading, setIsLoading] = useState(true);

useEffect(() => {
  const timer = setTimeout(() => setIsLoading(false), 300);
  return () => clearTimeout(timer);
}, []);
```
> Mirrors DashboardView pattern exactly. 300ms minimum display.

**TransactionsView render:**
```jsx
{isLoading
  ? Array.from({ length: 5 }).map((_, i) => <SkeletonTransactionRow key={i} />)
  : /* existing transaction list */
}
```

### Page Transitions (UX-07)

**Implementation — App.jsx view-routing block:**
```jsx
<AnimatePresence mode="wait">
  <motion.div
    key={view}
    initial={{ opacity: 0 }}
    animate={{ opacity: 1, transition: { duration: 0.15, ease: 'easeOut' } }}
    exit={{ opacity: 0, transition: { duration: 0.15, ease: 'easeIn' } }}
    className="flex-1 min-h-0 overflow-hidden"
  >
    {view === 'dashboard'     && <DashboardView ... />}
    {view === 'transactions'  && <TransactionsView ... />}
    {view === 'settings'      && <SettingsView ... />}
  </motion.div>
</AnimatePresence>
```

**No slide:** Views are not spatially ordered — fade only. 150ms is the minimum perceptible transition that communicates "content changed" without feeling sluggish.

**Note:** `AnimatePresence` is already imported in App.jsx (Phase 6). The page-transition `<AnimatePresence>` wraps the *view* block; the existing modal `<AnimatePresence>` wrappers are siblings, not nested.
